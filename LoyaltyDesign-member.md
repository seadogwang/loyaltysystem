# 会员管理界面设计文档（客服专用版）
**版本**：1.0
**状态**：待评审
**日期**：2026-06-06
**关联主文档**：《基于AI的下一代全渠道忠诚度管理SaaS平台 v7.1》
***
## 1. 设计目标
为客服人员提供**一站式会员查询与操作工作台**，支持通过手机号或会员ID快速检索，整合展示会员完整画像（基本信息、积分资产、交易流水、等级变更、渠道绑定），并支持必要的人工干预操作（信息修改、积分调整、等级调整、渠道解绑等）。
设计核心原则：
* **查询先行**：所有操作必须基于已查询到的会员，不存在独立操作入口。
* **信息聚合**：一屏内展示客服所需的所有会员信息，减少页面跳转。
* **轻量操作**：常用操作通过模态框完成，不离开当前页面。
* **安全审计**：所有人工操作记录操作日志，支持追溯。
***
## 2. 整体布局与交互流程
### 2.1 页面结构

```text
┌─────────────────────────────────────────────────────────────────────┐
│  [← 返回]  会员管理 / 客服工作台                        🔔  👤 客服名  │
├─────────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────────┐ │
│  │  🔍 会员查询                      [ 手机号/会员ID ]  [ 查询 ]    │ │
│  └─────────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌─ 会员摘要卡片 ──────────────────────────────────────────────────┐ │
│  │ 会员ID | 姓名 | 等级 | 状态 | 注册时间          │ 积分余额 | 信用 │ │
│  │ [信息修改] [调整积分] [调整等级] [冻结账户] [合并会员]            │ │
│  └─────────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌─ 积分账户详情 ──────────────────────────────────────────────────┐ │
│  │ 消费积分 | 成长值（含进度条） | 授信积分                         │ │
│  └─────────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  ┌─ Tab: 交易流水 | 等级变更日志 | 渠道绑定 ───────────────────────┐ │
│  │  [过滤器 + 表格（支持排序、过滤、溯源）]                         │ │
│  └─────────────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────────────┘
```
### 2.2 交互流程
1. 客服在查询输入框输入手机号（`MOBILE_PLAIN`）或会员ID，点击查询或按回车。
2. 系统校验租户权限，调用 `GET /api/members/search?keyword=xxx`。
3. 返回会员完整数据（基本信息、积分账户、最近交易流水、最近等级日志、渠道绑定列表）。
4. 依次渲染摘要卡片、积分账户、激活默认 Tab（交易流水）。
5. 客服可执行任意操作（信息修改、积分调整等），操作完成后自动刷新相关模块数据。
***
## 3. 查询区域
* 输入框宽度 300px，支持回车触发查询。
* 提供「最近查询」下拉提示（本地存储，最多 10 条），便于客服快速复用。
* 未查到会员时显示 Empty 状态，并提供「创建新会员」按钮（需二次确认，引导入会流程）。
***
## 4. 会员摘要卡片
### 4.1 布局与信息
采用左右两栏布局（左：身份信息，右：积分资产快照），白色卡片，1px 灰边，12px 圆角。
| 左侧信息            | 右侧信息         |
| --------------- | ------------ |
| 会员ID（可复制）       | 当前积分总余额（大字号） |
| 姓名、性别、生日        | 信用额度 / 已用信用  |
| 当前等级 + 等级图标     | -            |
| 状态标签（正常/冻结/合并中） | -            |
| 注册时间            | -            |
### 4.2 操作按钮组
卡片底部右对齐放置操作按钮（次按钮样式）：
* **信息修改**：打开模态框编辑会员基本信息及扩展属性。
* **调整积分**：打开模态框手动增减积分。
* **调整等级**：打开模态框手动升降等级。
* **冻结账户**：二次确认后冻结会员（禁止兑换，可入账）。
* **合并会员**：打开合并向导（见主文档 3.5）。
> 所有操作按钮仅在查询到会员后显示，未查询时隐藏。

以**水平卡片组**形式展示每种积分类型的核心指标：
| 消费积分      | 成长值                  | 授信积分      |
| --------- | -------------------- | --------- |
| 可用: 1,280 | 可用: 2,500            | 额度: 5,000 |
| 累计: 3,500 | 等级进度: ████████░░ 80% | 已用: 0     |
| 已兑: 2,220 | 下一等级阈值: 3,000        | 剩余: 5,000 |
| [查看明细]   | [查看明细]              | [查看明细]   |

* 「查看明细」跳转至交易流水 Tab 并预筛选对应积分类型。

## 6. 交易流水 Tab
### 6.1 表格列定义
| 列名   | 数据字段              | 是否支持排序 | 是否支持过滤  |
| ---- | ----------------- | ------ | ------- |
| 时间   | `createdAt`       | ✅      | ✅（时间范围） |
| 类型   | `transactionType` | ❌      | ✅（多选）   |
| 事件说明 | `description`     | ❌      | ❌       |
| 变动积分 | `amount`          | ✅      | ❌       |
| 账户余额 | `balanceAfter`    | ❌      | ❌       |
| 操作   | -                 | ❌      | ❌       |
**变动积分渲染规则**：
* 正数显示绿色 `+120`，负数显示红色 `-50`，零显示灰色 `0`。
### 6.2 排序与过滤
* **排序**：点击「时间」或「变动积分」表头，支持升序/降序切换（单列排序）。
* **过滤**：表格顶部提供过滤器行：
  * 交易类型：下拉多选（发放/兑换/过期/透支）。
  * 时间范围：快捷选项（今日/本周/本月/自定义）。
* 过滤条件变化后自动重新查询后端。
### 6.3 溯源操作
每行末尾提供「溯源」按钮（文字链接或图标）。点击后打开右侧抽屉（Drawer），展示该交易的积分批次明细：
* **发放交易**：显示发放批次详情（金额、过期时间、规则快照）。
* **兑换交易**：显示从 `redemption_allocation` 表关联的所有被消耗批次（按 FIFO 顺序排列），格式如下：
| 批次ID     | 发放时间       | 原始积分 | 本次消耗 | 批次剩余 |
| -------- | ---------- | ---- | ---- | ---- |
| BATCH-01 | 2026-05-01 | 30   | 30   | 0    |
| BATCH-02 | 2026-05-15 | 40   | 20   | 20   |
### 6.4 分页
默认每页 20 条，支持切换每页条数（20/50/100），后端游标分页或传统 `LIMIT/OFFSET`。

## 7. 等级变更日志 Tab
### 7.1 表格列定义
| 列名        | 数据字段                  | 排序 | 过滤      |
| --------- | --------------------- | -- | ------- |
| 变更时间      | `changedAt`           | ✅  | ✅（时间范围） |
| 原等级 → 新等级 | `oldTier` / `newTier` | ❌  | ❌       |
| 变更原因      | `changeReason`        | ❌  | ✅（多选）   |
| 关联事件      | `referenceEventId`    | ❌  | ❌       |
**变更原因枚举**：`UPGRADE`（升级）、`DOWNGRADE`（降级）、`CASCADE_DOWNGRADE`（级联降级）、`MERGE`（合并）、`MANUAL_UPGRADE`（人工升级）、`MANUAL_DOWNGRADE`（人工降级）。
### 7.2 过滤与排序
* 支持按「变更时间」排序。
* 支持按「变更原因」多选过滤。
* 支持按时间范围过滤。
### 7.3 数据来源
`tier_change_log` 表。

## 8. 渠道绑定 Tab
### 8.1 展示要求
按渠道分类展示会员的所有身份标识，便于客服快速定位。
### 8.2 界面设计
**方式一：表格（带渠道图标）**
| 渠道  | 标识类型   | 标识值             | 强标识 | 验证状态 | 绑定时间       | 操作 |
| --- | ------ | --------------- | --- | ---- | ---------- | -- |
| 天猫  | OUID   | tb123456        | ✅   | 已验证  | 2025-01-01 | 解绑 |
| 京东  | PIN    | jd_user_001   | ✅   | 已验证  | 2025-02-15 | 解绑 |
| 微信  | OpenID | oxc123...       | ✅   | 已验证  | 2025-03-10 | 解绑 |
| 抖音  | OpenID | dy123...        | ✅   | 已验证  | 2025-04-20 | 解绑 |
| POS | 手机号    | 138****5678 | ✅   | 已验证  | 2025-05-05 | 解绑 |
| 抖音  | 掩码手机   | 138****5678 | ❌   | 未验证  | 2025-04-20 | 验证 |
**方式二：折叠面板（按渠道分组）**
text
```
▼ 天猫
   - OUID: tb123456 (强标识，已验证)
   - 手机号(MD5): 6a7b... (弱标识)
▶ 京东
▶ 微信
```
推荐采用方式一，简单直观；若某会员渠道标识过多（如合并会员），可同时提供折叠面板作为备选。
### 8.3 操作说明
* **解绑**：仅允许对非唯一强标识（如渠道内 OpenID 且该会员还有其他登录方式）进行解绑。解绑前需二次确认。后端执行软删除（逻辑删除）。
* **验证**：针对抖音掩码手机，可触发重新验证流程（调用抖音接口，需用户授权，客服仅发起请求）。
* **手动绑定**：提供「绑定新渠道」按钮，客服输入渠道类型和标识值，调用绑定接口（需高级权限）。
### 8.4 数据来源
`member_unique_key` 表，按 `target_member_id` 查询，按渠道分组展示。

## 9. 详细操作规范
### 9.1 信息修改
**触发**：点击摘要卡片「信息修改」按钮。
**模态框**：宽度 720px，使用 Formily 动态表单渲染完整会员编辑界面（基础字段 + `ext_attributes` 动态字段）。
**保存逻辑**：
* 调用 `PUT /api/members/{memberId}`。
* 若修改手机号，后端需执行唯一性检查及可能的合并流程（见主文档 3.4）。
* 成功后关闭模态框，刷新会员摘要及扩展属性区域。
  **审计**：记录修改前后 JSON 快照。
### 9.2 调整积分
**触发**：点击「调整积分」按钮。
**模态框**：宽度 500px，表单包含：
* 积分类型（下拉：消费积分/成长值/授信积分）。
* 操作类型（增加/减少）。
* 积分数量（正整数）。
* 调整原因（必填）。
  **提交**：调用 `POST /api/members/{memberId}/points/adjust`。
  **异常处理**：若扣减时余额不足且积分类型不允许负数，返回错误提示。
  **刷新**：成功后关闭模态框，刷新积分账户卡片及交易流水列表。
  **审计**：记录调整前后余额、操作人、原因。
### 9.3 调整等级
**触发**：点击「调整等级」按钮。
**模态框**：宽度 480px，表单包含：
* 当前等级（只读）。
* 新等级（下拉，从 Program 等级列表取值）。
* 调整原因（必填）。
* 生效时间（可选，默认立即）。
  **提交**：调用 `POST /api/members/{memberId}/tier/adjust`。
  **降级二次确认**：若新等级低于当前等级，弹出二次警告。
  **刷新**：成功后关闭模态框，刷新会员摘要等级显示及等级日志 Tab。
  **审计**：记录等级变更原因 `MANUAL_UPGRADE` 或 `MANUAL_DOWNGRADE`，以及操作人。
### 9.4 冻结账户
**触发**：点击「冻结账户」按钮。
**二次确认**：警告“冻结后会员将无法使用积分兑换，是否继续？”
**提交**：调用 `POST /api/members/{memberId}/freeze`（后端设置 `frozen_status = 'FROZEN_REDEMPTION'`）。
**刷新**：更新会员卡片状态标签为“已冻结”。
### 9.5 合并会员
**触发**：点击「合并会员」按钮。
**流程**：打开合并向导（两步：输入目标会员ID → 预览差异 → 确认提交）。
**后端**：创建 `member_merge_task`，异步执行 Saga。
**前端**：轮询任务状态或通过 WebSocket 接收完成通知，完成后刷新当前会员信息（当前会员可能成为被合并方，需重新查询）。

## 10. API 汇总
| 方法   | 路径                                                       | 说明                          |
| ---- | -------------------------------------------------------- | --------------------------- |
| GET  | `/api/members/search?keyword=xxx`                        | 查询会员完整信息（含摘要、积分、最近流水/日志/渠道） |
| GET  | `/api/members/{memberId}/transactions`                   | 分页获取交易流水（支持排序、过滤）           |
| GET  | `/api/members/{memberId}/transactions/{txId}/allocation` | 获取交易关联的积分批次明细               |
| GET  | `/api/members/{memberId}/tier-logs`                      | 获取等级变更日志（支持分页、排序、过滤）        |
| GET  | `/api/members/{memberId}/channel-bindings`               | 获取渠道绑定列表                    |
| PUT  | `/api/members/{memberId}`                                | 修改会员信息                      |
| POST | `/api/members/{memberId}/points/adjust`                  | 手动调整积分                      |
| POST | `/api/members/{memberId}/tier/adjust`                    | 手动调整等级                      |
| POST | `/api/members/{memberId}/freeze`                         | 冻结账户                        |
| POST | `/api/members/merge`                                     | 创建会员合并任务                    |
所有接口均需携带 `X-Program-Code` Header，并校验租户权限及操作权限点。

## 11. 权限矩阵
| 操作   | 权限点                     | 默认角色    |
| ---- | ----------------------- | ------- |

| 解绑渠道 | `member:unbind_channel` | 高级客服    |

## 12. 前端技术实现要点
* **框架**：React 18 + TypeScript + Ant Design 5
* **状态管理**：Zustand（存储当前会员数据、查询关键词、分页状态）
* **数据请求**：react-query（缓存会员查询结果 5 分钟）
* **动态表单**：Formily 2.x（信息修改模态框）
* **表格**：Ant Design Table（支持排序、过滤、分页）
* **抽屉**：Ant Design Drawer（交易溯源）
* **图标**：自定义 SVG 纯线条图标（渠道图标、排序图标等）
***
## 13. 设计规范对齐（主文档第八章）
* **纯白底色**：页面背景 `#ffffff`，卡片边框 `#e8e8e8`。
* **黑线条风格**：按钮、图标、表头文字使用 `#1a1a1a`。
* **按钮样式**：主按钮黑底白字，次按钮白底黑框。
* **表格行**：`padding: 10px 0`，`border-bottom: 1px solid #e0e0e0`。
* **响应式**：表格支持横向滚动，最小宽度 1000px。
